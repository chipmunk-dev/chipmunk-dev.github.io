    재귀 함수

    재귀(recursion) 란 어떠한 문제가 있을 때 동일한 구조의 더 작은 문제로 나누고 작은 문제를 해결하여 전체를 해결하는 방법이다.

    자바에서 메서드를 작성하면서 또 다시 자기 자신을 호출하는 재귀 함수 를 작성할 수 있는데,
    재귀 함수를 잘 이용하면 반복적인 작업을 간결한 코드로 작성할 수 있다.

    재귀함수의 장점과 단점

    (장점)

    • 반복문 중첩을 사용하는 대신 코드가 간결해지고, 수정이 간편하다.
    • 변수를 적게 선언하여 사용할 수 있다.

    (단점)

    • 코드의 흐름을 반복문보다 파악하기 어렵다.
    • 재귀 함수는 함수가 끝나기 전에 자기자신을 반복적으로 호출하기에 메모리를 더 많이 사용한다.
    • 함수 호출과 함수 복귀를 위한 context switching 비용이 발생하고, 속도가 상대적으로 느린편이다.
    • 고려하지 못한 에지케이스가 있어 무한반복이 발생하면 프로그램이 터진다..

    이런 성능적인 단점은 꼬리 재귀 로 보완이 가능하다. 이 내용은 제일 아래에 정리하겠다.

    문제를 작게 나누고 가장 작은 문제로 만들기(base case)

    입력값을 예시로 [1, 2, 3, 4, 5] 값이 들어온다고 가정하자.
    이를 어떤방식으로 작게 생각할까?
    예를 들어 인자가 5개가 아니라 4개라면?
    [1, 2, 3, 4, 5]가 아니라 [2, 3, 4, 5]가 좀 더 작은 문제가 될 것이다.
    그리고 [2, 3, 4, 5] 보다는 [3, 4, 5]가 더 작고 결국 [] 빈 배열일 경우가 제일 작은 경우다.
    빈 배열은 요소가 없으니 합은 0이된다.
    이 처럼 제일 작은 문제가 정의되었을 때 이를 재귀에서는 탈출조건(base case)라고 한다.

    재귀적인 부분을 고민하기

    자 그렇다면 제일 작은케이스를 구하였지만, 해당 케이스를 어떻게 해결하면 좋을까?
    작은 케이스를 해결하게 되면 큰 문제는 쉽게 해결할 수 있다.

    위 처럼 첫번째 요소를 뺀 배열을 다시 재귀함수의 인자로 호출하는 것이다.
    이렇게 나누다보면 탈출 조건(base case)에 도달할 것이고 반환값들이 최상위 호출위치까지 돌아올 것이다.

    1. 함수의 입력값이 무엇이고 출력값이 어떻게 되는지 고민하기
    2. 문제를 작게 쪼개고 쪼개서 가장 작은 단위의 문제를 정의하기
    3. 작은 단위의 문제를 해결하기
    4. 작은 단위의 문제를 해결함으로써 남아있는 복잡한 문제를 해결하기

    꼬리 재귀

    꼬리재귀란?
    재귀는 좋은 이점이 많아 자주 사용되지만 성능적으로 부족하다.
    재귀의 장점은 살리면서 단점을 보완하는 방법 중 하나가 꼬리 재귀이다.
    재귀를 작성하다보면 반환할 때 연산을 하여 반환하는 경우가 많다.
    재귀 호출이 끝나면 아무 일도 하지 않고 결과만 바로 반환하도록 하는게 꼬리 재귀의 포인트다.
    이렇게 되면 이전 함수의 상태를 유지하지 않고 추가 연산을 하지도 않아 오버헤드를 해결할 수 있다.

    재귀하면 대표적인 예시로 사용되는 팩토리얼을 작성해보자.

    일반 재귀 방식

    인자값으로 5가 들어온다면

    기본적으로 함수가 재귀 함수를 호출하고, 그 결과값을 자기가 가지고 있던 값과 곱셈을 하여 다시 반환하는 그림을 가지고 있다.

    꼬리 재귀 방식

    return 부분을 보면 재귀 호출을 하되 호출을 완료한 재귀함수가 다른 연산없이 바로 리턴하는것을 볼 수 있다.

    인자값으로 5가 들어온다면

    반환 부분을 본다면 마지막에 계산된 120이라는 acc값이 그대로 전달된다.
    호출된 각 함수들이 반환되었을 때 별도의 연산없이 바로 전달한다.

    일반 재귀는 값을 반환 받았을 때 해당 값에 다른 연산을 하고 다시 다른 함수에 전달해준다.
    꼬리 재귀는 값을 반환 받으면 아무것도 하지 않고 그대로 반환한다.

    위의 차이점을 보았을 때 일반 재귀는 함수의 값을 반환받고 그 값을 다른 값과 연산을 해야하는 과정이 남아있기에 자기 자리를 기억해야한다. 그래서 스택에 해당 함수가 쌓일 때 함수가 받는 매개변수, 리턴값, 돌아갈 위치(자기 자리)를 기억한다.
    하지만 꼬리 재귀는 반환값에 따로 연산이 없으니 자기 자리를 기억하지 않아도 된다.
