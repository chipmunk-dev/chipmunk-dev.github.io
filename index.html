재귀 함수

재귀(recursion) 란 어떠한 문제가 있을 때 동일한 구조의 더 작은 문제로 나누고 작은 문제를 해결하여 전체를 해결하는 방법이다.

자바에서 메서드를 작성하면서 또 다시 자기 자신을 호출하는 재귀 함수 를 작성할 수 있는데,
재귀 함수를 잘 이용하면 반복적인 작업을 간결한 코드로 작성할 수 있다.

재귀함수의 장점과 단점

(장점)

  • 반복문 중첩을 사용하는 대신 코드가 간결해지고, 수정이 간편하다.
  • 변수를 적게 선언하여 사용할 수 있다.

(단점)

  • 코드의 흐름을 반복문보다 파악하기 어렵다.
  • 재귀 함수는 함수가 끝나기 전에 자기자신을 반복적으로 호출하기에 메모리를 더 많이 사용한다.
  • 함수 호출과 함수 복귀를 위한 context switching 비용이 발생하고, 속도가 상대적으로 느린편이다.
  • 고려하지 못한 에지케이스가 있어 무한반복이 발생하면 프로그램이 터진다..

이런 성능적인 단점은 꼬리 재귀 로 보완이 가능하다. 이 내용은 제일 아래에 정리하겠다.

재귀적으로 사고하기

하나의 문제를 재귀적으로 해결하는 방법을 알아보자.

위와 같은 문제가 있다고 하고, 이를 재귀적으로 생각해보자.

문제를 작게 나누고 가장 작은 문제로 만들기(base case)

입력값을 예시로 [1, 2, 3, 4, 5] 값이 들어온다고 가정하자.
이를 어떤방식으로 작게 생각할까?
예를 들어 인자가 5개가 아니라 4개라면?
[1, 2, 3, 4, 5]가 아니라 [2, 3, 4, 5]가 좀 더 작은 문제가 될 것이다.
그리고 [2, 3, 4, 5] 보다는 [3, 4, 5]가 더 작고 결국 [] 빈 배열일 경우가 제일 작은 경우다.
빈 배열은 요소가 없으니 합은 0이된다.
이 처럼 제일 작은 문제가 정의되었을 때 이를 재귀에서는 탈출조건(base case)라고 한다.
이 base case에 만족하면 더이상 재귀적인 호출은 없고 다시 최초 호출시점으로 값을 반환시작한다.

재귀적인 부분을 고민하기

자 그렇다면 제일 작은케이스를 구하였지만, 해당 케이스를 어떻게 해결하면 좋을까?
작은 케이스를 해결하게 되면 큰 문제는 쉽게 해결할 수 있다.

위 처럼 첫번째 요소를 뺀 배열을 다시 재귀함수의 인자로 호출하는 것이다.
이렇게 나누다보면 탈출 조건(base case)에 도달할 것이고 반환값들이 최상위 호출위치까지 돌아올 것이다.

해결된 문제와 그 다음 작은 문제롤 통합하여 해결하고 해결하고.. 이런식으로 큰 문제를 해결할 수있다.

  2. 문제를 작게 쪼개고 쪼개서 가장 작은 단위의 문제를 정의하기
  3. 작은 단위의 문제를 해결하기
  4. 작은 단위의 문제를 해결함으로써 남아있는 복잡한 문제를 해결하기

꼬리 재귀

꼬리재귀란?
재귀는 좋은 이점이 많아 자주 사용되지만 성능적으로 부족하다.
재귀의 장점은 살리면서 단점을 보완하는 방법 중 하나가 꼬리 재귀이다.
재귀를 작성하다보면 반환할 때 연산을 하여 반환하는 경우가 많다.
재귀 호출이 끝나면 아무 일도 하지 않고 결과만 바로 반환하도록 하는게 꼬리 재귀의 포인트다.
이렇게 되면 이전 함수의 상태를 유지하지 않고 추가 연산을 하지도 않아 오버헤드를 해결할 수 있다.

재귀하면 대표적인 예시로 사용되는 팩토리얼을 작성해보자.

호출된 각 함수들이 반환되었을 때 별도의 연산없이 바로 전달한다.

일반 재귀는 값을 반환 받았을 때 해당 값에 다른 연산을 하고 다시 다른 함수에 전달해준다.
꼬리 재귀는 값을 반환 받으면 아무것도 하지 않고 그대로 반환한다.

위의 차이점을 보았을 때 일반 재귀는 함수의 값을 반환받고 그 값을 다른 값과 연산을 해야하는 과정이 남아있기에 자기 자리를 기억해야한다. 그래서 스택에 해당 함수가 쌓일 때 함수가 받는 매개변수, 리턴값, 돌아갈 위치(자기 자리)를 기억한다.
하지만 꼬리 재귀는 반환값에 따로 연산이 없으니 자기 자리를 기억하지 않아도 된다.

컴퓨터와 프로그래밍의 이해

목차

  • 컴퓨터의 이해

컴퓨터의 이해

컴퓨터란 무엇인가?

입력 받은 데이터를 정해진 **규칙**대로 처리해서 처리 결과를 저장하고 출력하는 전자 장치

정해진 규칙대로 → 소프트웨어
입력받은 데이터를 처리하고 그 결과를 출력하는 장치 → 하드웨어

컴퓨터는 소프트웨어와 하드웨어로 구성되어있으며, 이 둘의 상호작용을 통해 동작한다.

소프트웨어

소프트웨어는 하드웨어가 어떻게 동작해야 하는지 규정해놓은 명령어의 집합이다.
이는 크게 시스템소프트웨어 와 응용소프트웨어 로 분류된다.

시스템 소프트웨어
응용 소프트웨어를 실행하기 위한 플랫폼 역활을 수행하며, 하드웨어와 직접적으로 소통한다.

  • 응용소프트웨어의 실행환경
    • Mac, Windows, Linux과 같은 운영체제 및 런타임 환경(Node, JRE)
  • 하드웨어 장치를 관리
    • 키보드 드라이버, 마우스 드라이버, USB 드라이버와 같은 장치 드라이버들

응용 소프트웨어
시스템 소프트웨어를 제외한 모든 소프트웨어를 의미, 애플리케이션/앱 이라고도 하며, 프로그램이라고도 불린다.

하드웨어

컴퓨터를 구성하는 기계적 장치들을 말한다.
CPU, Memory, Graphic 등의 물리적인 장치들이다.
컴퓨터를 구성하는 하드웨어는 중앙처리장치, 기억장치, 입력장치, 출력장치로 분류된다.
그리고 각 장치는 시스템 버스로 연결되어 있다.

중앙처리장치 (Central Processing Unit, CPU)
CPU는 프로그램을 실행, 데이터 처리를 담당하는 핵심 장치이며 프로세서라고도 한다.
제어장치, 연산장치, 레지스터로 구성되어 있다.

  • 제어장치
    • CPU의 동작과 데이터 흐름을 관리한다.
    • 운영체제와 소통하며, 자원을 관리한다.
  • 연산장치
    • 프로그램의 명령어를 수행한다.
    • 코드를 실질적으로 실행시키는 장치
  • 레지스터
    • CPU 안에 있는 작고 빠른 기억장치 → CPU와 가장 가까운 기억장치
    • 연산에 필요한 데이터들을 임시적으로보관하고 제공한다.

기억장치 (Memory Device)
CPU 동작에 필요한 데이터들을 단기 또는 장기적으로 보관한다.
단기적으로 기억하는 것은 주기억장치이며, 장기적으로 기억하는 것은 보조기억장치이다.

  • 보조기억장치
    • 컴퓨터에 필요한 정보를 반영구적으로 저장
    • 큰 용량의 데이터를 저장할 수 있지만 데이터의 일기/쓰기 속도가 상대적으로 느리다.
    • CPU의 연산속도가 엄청 빠른편이나 보조기억장치는 느리기 떄문에 직접적으로 소통하지 않는다.
    • SSD(Solid State Drive), HDD(Hard Disk Drive)가 있다.
  • 주기억장치
    • 컴퓨터가 프로그램을 수행하는 동안 필요한 모든 데이터들을 저장
    • 보조기억장치와 같이 큰 용량을 저장할수는 없지만 데이터의 읽기/쓰기 속도가 빠른편이다.
    • 빠른 연산 속도로 인하여 CPU와 직접적으로 소통한다.
    • RAM(Random Access Memory), ROM(Read Only Memory)가 있다.

캐시메모리
캐시메모리는 CPU가 사용한 데이터 중에서 재사용성이 높은 데이터를 주기억장치로부터 미리 저장한다.
주기억장치가 빠르다지만 CPU만큼은 아니기에 캐시메모리를 통해 데이터들을 좀더 빠르고 효율적으로 저장하고 가져올 수 있다.

기억장치의 계층구조
레지스터 → 캐시메모리 → 주기억장치 → 보조기억장치
우측으로 갈수록 메모리의 가격이 저렴하며, 용량을 많이 저장할 수 있다.
하지만 반대로 속도는 느려진다.

입력 장치 (Input Device)
사용자로부터 컴퓨터에 데이터를 입력받는 장치 → 키보드, 마우스,마이크 등

출력 장치 (Output Device)
사용자에게 사람이 감각할 수 있는 형태로 데이터를 출력해주는 장치 → 모니터, 스피커, 프린터 등

시스템 버스(System Bus)
하드웨어 장치간의 통로
각각의 장치를 물리적으로 연결한 전선을 의미하기에 시스템 버스도 하드웨어에 포함된다.

  • 데이터 버스
    • 하드웨어 간에 데이터를 전달하는 통로
  • 주소 버스
    • 데이터가 어디에 도착해야 하는지에 대한 정보를 전달하는 통호
  • 제어 버스
    • CPU의 제어장치가 생성한 제어 신호를 다른 장치로 전달하는 통로

프로그래밍의 이해

컴퓨터에게 문제 해결을 맡기는 커뮤니케이션

프로그램을 만드는 과정, 특정 목적을 달성하기 위해 설계된 알고리즘을
프로그래밍 언어를 사용하여 코드로 작성하는 과정

프로그램

프로그래밍을 통하여 만들어진 결과물
즉, 어떠한 목적을 달성하기 위한 컴퓨터 명령문의 집합체

프로그래밍 언어

컴퓨터는 사람들의 언어를 이해하지 못한다.
그러기에 컴퓨터가 이해할 수 있는 언어로 설명해주어야 한다.
컴퓨터는 기계어를 사용한다. 사람들은 사람들만의 언어를 사용하는데,
이 둘의 매개해주는 언어가 프로그래밍 언어다.
사람들의 명령어를 이해하고 이를 통하여 컴퓨터를 동작하게 한다.

컴파일

프로그래밍 언어는 사람의 언어와 컴퓨터 언어를 매개하지만,
컴퓨터는 프로그래밍 언어로 작성된 소스코드를 이해하지 못한다.
이를 컴퓨터에게 이해시키기 위해 코드를 변환하는 작업을 하는데 이를
**컴파일(Compile)**이라고 한다.
자바는 컴파일을 통하여 소스코드를 byte코드로 변환하여 JVM에게 던져준다.

빌드

작성한 소스코드를 실행할 수 있는 결과물로 변환시키는 과정을 빌드라고한다.
컴파일은 빌드과정의 일부이다. 컴파일을 마친 코드는 기계어 코드로 변환하고 이를 통하여 실행할 수 있는 .exe, .msi등 확장자를 가진 실행 파일로 만든다. 이런 빌드가 완료되어야 실행 가능한 프로그램이 완성된다.

통합 개발환경 (IDE)

Integrated Development Environment(이하 IDE)는 코드를 작성할 때 사용하는 프로그램으로, 프로그래밍에 필요한 모든 작업을 하나의 프로그램으로 처리할 수 있도록 도와준다.
코드를 작성하고 편집할 수 있으며, 디버깅과 빌드도 IDE에서 가능하다.
대표적으로 Visual Studio Code, IntelliJ, Eclipse등이 있다.

해당 포스팅은 코드라뗴의 자바 강의를 공부하며 작성한 글입니다.

리터럴과 메모리의 관계

정수형에서 리터럴과 실수형에서 리터럴에는 접미사가 붙는 경우가 있다.
long 타입과 float 타입의 리터럴은 L과 F을 붙인다.

Java는 정수형은 int를 실수형은 double을 기본 리터럴로 사용한다.
그렇기 때문에 변수에 리터럴을 할당할 때에도 메모리의 변환이 일어나는데
아래의 코드를 보면서 확인하자.

num이라는 변수에 10을 할당할 때 메모리에서는 여러가지 일이 일어나는데

  1. int 자료형인 num 변수가 메모리에 4byte를 할당한다.
  2. 오른쪽에 10이라는 숫자는 스택 메모리라는 공간에 일시적으로 4byte를 할당하여 00000000 00000000 00000000 00001010 이라는 값을 적재한다.
  3. 그런 다음 변수의 메모리 공간에 리터럴 값이 저장되고, 리터럴이 저장된 메모리 공간을 해제한다.

그러면 byte와 short를 할당할 때는 어떨까?

short에 정수 리터럴 100을 할당할 때는

  1. short 자료형인 num 변수가 메모리에 2byte를 할당한다.
  2. 리터럴 100은 정수형(4byte)이기에 00000000 00000000 00000000 01100100을 스택 메모리에 할당한다.
  3. 리터럴값은 4byte이고, num은 2byte(short)이기에 리터럴 메모리의 앞 2byte를 제거한 뒤(00000000 01100100)
    변수 메모리에 할당한다. 그런 다음 리터럴이 저장된 스택 메모리는 공간을 해제한다.

만약 리터럴에 저장된 값이 2byte보다 클 경우 명시적 형변환을 해야한다. (형변환은 다른 포스트에서 작성해보겠다)

그렇다면 정수 리터럴인 4byte보다 큰 8byte(long)에 값을 할당될 때는 어떻게 될까?
이건 long타입의 변수를 선언하기에 jvm(자바 가상 머신)에게 리터럴 값을 8byte로 할당해달라고 L접미사를 사용한다.

위 처럼 리터럴에 L을 붙여줌으로 Java에게 큰 값이다라는 정보를 준다.

  1. long 자료형인 num 변수가 메모리에 8byte 메모리 공간을 확보한다.
  2. 리터럴 1000조는 큰 수이기에 접미사 L을 통하여 스택의 8Byte공간에 리터럴 값을 할당한다.
  3. 리터럴값을 num 변수에 리터럴값을 num변수에 저장(복사)후 스택 메모리를 해제한다.

실수 리터럴중 32.123456789F와 32.123456789은 다르다.

두 값은 실수의 2진수 표현에 대하여 포스팅에서 다뤘듯이 단정도와 배정도의 차이로 인하여 2진수의 표현방식이 다르며, 실제 값도 다르게 표현된다.

bigRealNumber1은 단정도로 실수를 2진수로 변환 후 8byte로 암시적 형변환이 되는것이라
0100000001000000000011111100110101100000000000000000000000000000 지수부와 가수부의 추가공간이 다 0으로 채워지지만

bigRealNumber2는 배정도로 실수를 2진수로 변환하기 때문에 8byte의 지수부와 가수부를 다 사용하여 저장한다.
0100000001000000000011111100110101101110100110111001110010110010

그렇기에 보기에는 같은 수이지만, 메모리에는 실제적으로 다른 값으로 저장된다.

상수

자바에는 두가지 상수타입이 존재한다.

리터럴(literal)

자바에서 리터럴 혹은 리터럴 값이라고 많이 표현하는데

리터럴은 상수이며 변할 수 없다.

심볼릭 상수는 변수와 final키워드가 같이 사용된 형태를 말한다.

실수의 정밀도

실수의 정밀도란, 실수를 변수에 저장했을 때 이 값이 얼마나 정확한지 나타내는것을 정밀도라고 한다.

아래에서 알아보겠지만 프로그래밍에서 실수를 저장할때 메모리를 효율적으로 사용하기 위해 부동소수점방식을 이용한다.

이 방식은 무한한 실수를 효율적으로 메모리에 저장하는 대신에 변수에 저장된 실수가 원본 값과 100% 일치하지 않는 다는 단점이 있다. 그래서 실수의 정밀도라는 개념이 존재한다.

Float일 경우에는 6자리까지 정확한 값을 나타내고(약 6 ~ 7자리)
Double일 경우에는 15자리까지 정확한 값을 나타낸다(15 ~ 16자리).
Double형이 메모리 공간이 큰 만큼 정밀도가 높다.
그래서 Float보단 Double을 사용하는것이 낫다.

맞다.. 가급적 오차가 중요하지 않거나 정밀도가 보장된 범위 내에서 사용하는것이 좋다고 한다.

고정소수점과 부동소수점

실수를 2진수로 표현하는데는 고정소수점과 부동소수점이 있다.

4byte(Float) 고정소수점

첫번째 자리의 부호비트를 제외한 31bit중에서
정수를 표현하는 15bit와 소수를 표현하는 16비트로 구성된다.

이는 이름과 같이 고정되어있는 영역이라, 정수에서 사용되지 않는 비트는 낭비가 되고, 큰 수일수록 많은 메모리가 필요하다.

4byte(Float) 부동소수점

이를 해결하기위해 부동소수점이라는 방식을 사용한다.
4byte(32bit)의 부동소수점은 단정도라고 한다.

첫번째 자리는 부호비트이며 8bit의 지수부와 32bit의 가수부로 구성되어있으며, 실수를 지수부와 가수부로 나누어 저장한다.

똑같이 앞 1bit는 부호를 나타내며, 지수부 11bit와 가수부 52bit로 구성되어있다.

14.14를 단정도(Float) 부동소수점 방식으로 저장하기

14.14라는 실수를 단정도 방식으로 변환되는 과정을 알아보자.

  1. 14.14에서 정수 추출
    14.14 -> 14
  2. 정수를 2진수로 변환
    14 -> 1110
  3. 소수부분을 추출하여 가수부 비트 수(23)까지 2를 곱하며 2진수 추출
    3.1 곱한값이 1이될때까지 반복하며, 1이 넘어갔을 때 1을 감소시킨 다음 다시 2를 곱해나가는 방식이다.
  • 0.14 * 2 = 0.28 -> (1보다 작아 0)
  • 0.28 * 2 = 0.56 -> (0)
  • 0.56 * 2 = 1.12 -> (1 이상이라 1을 감소시키고 이진수 1 표현)
  • 0.12 * 2 = 0.24 -> (0)
  • 0.24 * 2 = 0.48 -> (0)
  • 0.48 * 2 = 0.96 -> (0)
  • 0.96 * 2 = 1.92 -> (1)
  • 0.92 * 2 = 1.84 -> (1)
  • 0.84 * 2 = 1.68 -> (1)
  • 0.68 * 2 = 1.32 -> (1)
  • 0.32 * 2 = 0.64 -> (0)
  • 0.64 * 2 = 1.28 -> (1)
  • 0.28 * 2 = 0.56 -> (0)
  • 0.56 * 2 = 1.12 -> (1)
  • 0.12 * 2 = 0.24 -> (0)
  • 0.24 * 2 = 0.48 -> (0)
  • 0.48 * 2 = 0.96 -> (0)
  • 0.96 * 2 = 1.92 -> (1)
  • 0.92 * 2 = 1.84 -> (1)
  • 0.84 * 2 = 1.68 -> (1)
  • 0.68 * 2 = 1.36 -> (1)
  • 0.36 * 2 = 0.72 -> (0)
  • 0.72 * 2 = 1.44 -> (1)

23번 자리까지 표현했을 떄 00100011110101000111101 이 나온다.
이를 정수부와 소수부를 연결했을 때
1110.00100011110101000111101이 된다.

  1. 맨 앞자리까지 소수점을 이동시킨다.
    1.11000100011110101000111101 (3자리 이동)

5.앞의 1을 제거한다.
11000100011110101000111101(2)

  1. 소수점을 이동하며 소수부가 23자리 이상이 되었는데 넘어간 자리수는 제거(3자리)
    11000100011110101000111(2)

  2. 이동한 소수점 자리수 + 127(32bit Bias) 계산
    3 + 127 = 130

  3. 130 2진수 표기
    10000010(2)

  4. 계산된 비트를 연결하기
    부호비트 0
    지수비트 10000010
    가수비트 11000100011110101000111

2진수와 정수의 계산에 대하여

정수와 실수

자바에서 정수와 실수를 표현하는 기본 자료형에는 4가지가 있다.

정수

  • (1byte) byte
  • (2byte) short
  • (4byye) int
  • (8byte) long

실수

  • (4byte) float
  • (8byte) double

정수와 실수는 메모리에 자료형의 크기만큼 공간을 할당하여 데이터를 2진수로 저장한다.

그렇다면, 자바에서 정수와 실수가 어떻게 저장되고 변화될까?

부호비트

우선 정수나 실수 모두 메모리에는 2진수로 변환하여 저장한다.
1byte = 8bit로 표현하는데, 이미지화를 시킨다면
1byte = 00000000와 같이 8개의 비트로 나타낸다.

이 중에서 첫번째 자리의 비트는 부호비트로 사용한다.
부호비트란 양수와 음수를 표현하기 위한 비트로써
0일때는 양수
1일때는 음수
로 표현한다.

부호비트를 제외한 나머지 비트는 정수를 표현할 때 사용하며, 이를 정수부라고 표현한다.

음수를 만드는 방법

사람에게 빼기에 대한 계산식을 적어보라고한다면

이렇게 식을 적고 계산한다.
하지만 컴퓨터에게는 빼기가아닌 이또한 더하기 연산으로 계산한다.

수를 빼는게 아닌 양수에 음수를 더한다고 보면된다.

그러면 빼기 연산자가 실행되는 과정을 살펴보자.

10 - 5라는 식이 있다고 봤을 때
(1byte 기준으로 작성)

  1. 10을 2진수로 변경한다
    1.1 10 -> 00001010
  2. -5를 2진수로 변경한다.
    2.1 5의 2진수 -> 00000101
    2.2 5의 2진수에 1의보수(0은 1로, 1은 0으로)
    00000101 -> 11111010
    2.3 반전시킨 1의 보수에 1비트 가산(2의보수)
    11111011
    3.두 수를 더하기 연산
    00001010 + 11111011

결과: 00000101 -> 5

위처럼 1의 보수와 2의 보수를 이용하여 양수를 음수로 변환 후 더하기 연산으로 값을 도출한다.
