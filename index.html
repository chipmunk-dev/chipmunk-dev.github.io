변수의 스코프와 라이프사이클

Posted on 2022-09-21 In Java Disqus: Symbols count in article: 2.3k Reading time ≈ 2 mins.

어떤 프그래밍 언어를 사용하든, 데이터를 저장할 수 있는 변수(variable) 개념이 있다.
자바에서는 이러한 변수를 선언하는 위치에 따라 4가지로 구분할 수 있다.

지역 변수 (Local Variable)

지역 변수는 중괄호 내에서 선언된 변수를 뜻한다.
선언된 중괄호 내에서만 유효하며(생명주기),
같은 이름의 변수가 다른 중괄호에 선언되어도 서로 다른 값으로 사용된다.

위 코드를컴파일 하면 test2 메서드의 출력 코드에서 에러가 발생한다.

다른 경우를 살펴보자

위 케이스와 같지만 이번에는 중괄호가 메서드의 중괄호가 아닌 조건문의 중괄호다.
지역변수는 중괄호에 영향을 받으니 컴파일을 하여 출력한다면, else의 분기문으로 들어가
해당 지역에 선언된 age의 값을 찾아 18을 출력한다!



매개 변수 (Parameters)

매개 변수는 메소드에 넘겨주는 변수다.
생명주기는 해당 메소드가 호출될 때 시작되며, 메소드가 종료되면 소멸된다(가비지 컬렉터)

위 코드에서 printNumber라는 메소드에 parameter매개변수를 받아서 사용한다.
이 변수는 해당 메소드에서만 유효하며, 호출하는 메인 메소드에서 호출하면 에러가 발생한다.



인스턴스 변수 (Instance Variable)

인스턴스 변수는 클래스내에서 메소드 밖에, 클래스 안에 선언된 변수이며, static키워드가 없어야 한다.
생명주기는 객체가 생성될 때 시작되고, 그 객체의 참조가 없어서 소멸될 때 같이 사라진다.
접근을 할 때는 객체.변수로 접근이 가능하며, 캡슐화가 되었을 경우 메서드를 이용해 접근할 수 있다(접근 못할 수도 있음)

인스턴스 변수로 선언된 instanceVariable은 해당 객체가 생성될 때 생명이 시작되며,
iv.intanceVariable을 통하여 접근할 수 있다.
iv에 대한 참조가 사라진다면 해당 변수도 생명주기가 끝나며 더 이상 사용할 수 없다.



클래스 변수 (Class Variable)

클래스 변수는 인스턴스 변수와 같지만, 선언된 변수 앞에 static키워드가 존재한다.
이는 클래스가 처음 호출될 때 생명이 시작되고, 자바 프로그램이 종료될 때 사라진다.
클래스 변수는 인스턴스를 생성하지 않더라도 Class명.class변수로 접근할 수 있다.

리터럴과 메모리

Posted on 2022-09-20 Edited on 2022-09-21 In Java Disqus: Symbols count in article: 1.4k Reading time ≈ 1 mins.

해당 포스팅은 코드라뗴의 자바 강의를 공부하며 작성한 글입니다.

리터럴과 메모리의 관계

정수형에서 리터럴과 실수형에서 리터럴에는 접미사가 붙는 경우가 있다.
long 타입과 float 타입의 리터럴은 L과 F을 붙인다.

Java는 정수형은 int를 실수형은 double을 기본 리터럴로 사용한다.
그렇기 때문에 변수에 리터럴을 할당할 때에도 메모리의 변환이 일어나는데
아래의 코드를 보면서 확인하자.

num이라는 변수에 10을 할당할 때 메모리에서는 여러가지 일이 일어나는데

  1. int 자료형인 num 변수가 메모리에 4byte를 할당한다.
  2. 오른쪽에 10이라는 숫자는 스택 메모리라는 공간에 일시적으로 4byte를 할당하여 00000000 00000000 00000000 00001010 이라는 값을 적재한다.
  3. 그런 다음 변수의 메모리 공간에 리터럴 값이 저장되고, 리터럴이 저장된 메모리 공간을 해제한다.

그러면 byte와 short를 할당할 때는 어떨까?

short에 정수 리터럴 100을 할당할 때는

  1. short 자료형인 num 변수가 메모리에 2byte를 할당한다.
  2. 리터럴 100은 정수형(4byte)이기에 00000000 00000000 00000000 01100100을 스택 메모리에 할당한다.
  3. 리터럴값은 4byte이고, num은 2byte(short)이기에 리터럴 메모리의 앞 2byte를 제거한 뒤(00000000 01100100)
    변수 메모리에 할당한다. 그런 다음 리터럴이 저장된 스택 메모리는 공간을 해제한다.

만약 리터럴에 저장된 값이 2byte보다 클 경우 명시적 형변환을 해야한다. (형변환은 다른 포스트에서 작성해보겠다)

그렇다면 정수 리터럴인 4byte보다 큰 8byte(long)에 값을 할당될 때는 어떻게 될까?
이건 long타입의 변수를 선언하기에 jvm(자바 가상 머신)에게 리터럴 값을 8byte로 할당해달라고 L접미사를 사용한다.

위 처럼 리터럴에 L을 붙여줌으로 Java에게 큰 값이다라는 정보를 준다.

  1. long 자료형인 num 변수가 메모리에 8byte 메모리 공간을 확보한다.
  2. 리터럴 1000조는 큰 수이기에 접미사 L을 통하여 스택의 8Byte공간에 리터럴 값을 할당한다.
  3. 리터럴값을 num 변수에 리터럴값을 num변수에 저장(복사)후 스택 메모리를 해제한다.

실수 리터럴중 32.123456789F와 32.123456789은 다르다.

두 값은 실수의 2진수 표현에 대하여 포스팅에서 다뤘듯이 단정도와 배정도의 차이로 인하여 2진수의 표현방식이 다르며, 실제 값도 다르게 표현된다.

bigRealNumber1은 단정도로 실수를 2진수로 변환 후 8byte로 암시적 형변환이 되는것이라
0100000001000000000011111100110101100000000000000000000000000000 지수부와 가수부의 추가공간이 다 0으로 채워지지만

bigRealNumber2는 배정도로 실수를 2진수로 변환하기 때문에 8byte의 지수부와 가수부를 다 사용하여 저장한다.
0100000001000000000011111100110101101110100110111001110010110010

그렇기에 보기에는 같은 수이지만, 메모리에는 실제적으로 다른 값으로 저장된다.

Java의 두 가지 상수

Posted on 2022-09-20 In Java Disqus: Symbols count in article: 461 Reading time ≈ 1 mins.

상수

프로그래밍에서 변할 수 없는 수를 상수라고 한다.
자바에는 두가지 상수타입이 존재한다.

리터럴(literal)

자바에서 리터럴 혹은 리터럴 값이라고 많이 표현하는데
정수: 100, 500, 3 …
실수: 3.14, 2.111111 …
문자: ‘Z’, ‘오’ …
문자열: “배고파”, “안녕” …
논리: true, false

등 값 그 자체를 리터럴이라고 한다.

100과 500은 정수 리터럴이며, 3.14, 2.11111 은 실수 리터럴이다.
리터럴은 상수이며 변할 수 없다.

심볼릭(Symbolic)

심볼릭 상수는 변수와 final키워드가 같이 사용된 형태를 말한다.

위처럼 앞에 final을 상징적으로 붙여 “ 이 변수는 앞으로 상수야! “ 라고 표현해줄 수 있다.

다만, 초기화를 하지 않고 선언했을 때 값을 한번 할당할 수 있다.

이번에는 실수를 2진수로 표현하는 방법과 그 과정에 대하여 공부하며 작성한다.

  • Double

실수의 정밀도

실수의 정밀도란, 실수를 변수에 저장했을 때 이 값이 얼마나 정확한지 나타내는것을 정밀도라고 한다.

아래에서 알아보겠지만 프로그래밍에서 실수를 저장할때 메모리를 효율적으로 사용하기 위해 부동소수점방식을 이용한다.

이 방식은 무한한 실수를 효율적으로 메모리에 저장하는 대신에 변수에 저장된 실수가 원본 값과 100% 일치하지 않는 다는 단점이 있다. 그래서 실수의 정밀도라는 개념이 존재한다.

Float일 경우에는 6자리까지 정확한 값을 나타내고(약 6 ~ 7자리)
Double일 경우에는 15자리까지 정확한 값을 나타낸다(15 ~ 16자리).
Double형이 메모리 공간이 큰 만큼 정밀도가 높다.
그래서 Float보단 Double을 사용하는것이 낫다.

엥? 그런데 실수를 사용하면 오차가 생겨서 위험한거 아닌가?

맞다.. 가급적 오차가 중요하지 않거나 정밀도가 보장된 범위 내에서 사용하는것이 좋다고 한다.

고정소수점과 부동소수점

실수를 2진수로 표현하는데는 고정소수점과 부동소수점이 있다.

4byte(Float) 고정소수점

첫번째 자리의 부호비트를 제외한 31bit중에서
정수를 표현하는 15bit와 소수를 표현하는 16비트로 구성된다.

이는 이름과 같이 고정되어있는 영역이라, 정수에서 사용되지 않는 비트는 낭비가 되고, 큰 수일수록 많은 메모리가 필요하다.

4byte(Float) 부동소수점

이를 해결하기위해 부동소수점이라는 방식을 사용한다.
4byte(32bit)의 부동소수점은 단정도라고 한다.

첫번째 자리는 부호비트이며 8bit의 지수부와 32bit의 가수부로 구성되어있으며, 실수를 지수부와 가수부로 나누어 저장한다.

8byte(Double) 부동소수점

8byte(64bit) 부동소수점방식은 배정도라고 한다.
똑같이 앞 1bit는 부호를 나타내며, 지수부 11bit와 가수부 52bit로 구성되어있다.

14.14를 단정도(Float) 부동소수점 방식으로 저장하기

14.14라는 실수를 단정도 방식으로 변환되는 과정을 알아보자.

  1. 14.14에서 정수 추출
    14.14 -> 14
  2. 정수를 2진수로 변환
    14 -> 1110
  3. 소수부분을 추출하여 가수부 비트 수(23)까지 2를 곱하며 2진수 추출
    3.1 곱한값이 1이될때까지 반복하며, 1이 넘어갔을 때 1을 감소시킨 다음 다시 2를 곱해나가는 방식이다.
  • 0.14 * 2 = 0.28 -> (1보다 작아 0)
  • 0.28 * 2 = 0.56 -> (0)
  • 0.56 * 2 = 1.12 -> (1 이상이라 1을 감소시키고 이진수 1 표현)
  • 0.12 * 2 = 0.24 -> (0)
  • 0.24 * 2 = 0.48 -> (0)
  • 0.48 * 2 = 0.96 -> (0)
  • 0.96 * 2 = 1.92 -> (1)
  • 0.92 * 2 = 1.84 -> (1)
  • 0.84 * 2 = 1.68 -> (1)
  • 0.68 * 2 = 1.32 -> (1)
  • 0.32 * 2 = 0.64 -> (0)
  • 0.64 * 2 = 1.28 -> (1)
  • 0.28 * 2 = 0.56 -> (0)
  • 0.56 * 2 = 1.12 -> (1)
  • 0.12 * 2 = 0.24 -> (0)
  • 0.24 * 2 = 0.48 -> (0)
  • 0.48 * 2 = 0.96 -> (0)
  • 0.96 * 2 = 1.92 -> (1)
  • 0.92 * 2 = 1.84 -> (1)
  • 0.84 * 2 = 1.68 -> (1)
  • 0.68 * 2 = 1.36 -> (1)
  • 0.36 * 2 = 0.72 -> (0)
  • 0.72 * 2 = 1.44 -> (1)

23번 자리까지 표현했을 떄 00100011110101000111101 이 나온다.
이를 정수부와 소수부를 연결했을 때
1110.00100011110101000111101이 된다.

  1. 맨 앞자리까지 소수점을 이동시킨다.
    1.11000100011110101000111101 (3자리 이동)

5.앞의 1을 제거한다.
11000100011110101000111101(2)

  1. 소수점을 이동하며 소수부가 23자리 이상이 되었는데 넘어간 자리수는 제거(3자리)
    11000100011110101000111(2)

  2. 이동한 소수점 자리수 + 127(32bit Bias) 계산
    3 + 127 = 130

  3. 130 2진수 표기
    10000010(2)

  4. 계산된 비트를 연결하기
    부호비트 0
    지수비트 10000010
    가수비트 11000100011110101000111

연결비트 01000001011000100011110101000111 이 된다.

배정도(8Byte)또한 변환 방식이 같다.

2진수와 정수의 계산에 대하여

Posted on 2022-09-19 In Java Disqus: Symbols count in article: 858 Reading time ≈ 1 mins.

정수와 실수

자바에서 정수와 실수를 표현하는 기본 자료형에는 4가지가 있다.

정수

  • (1byte) byte
  • (2byte) short
  • (4byye) int
  • (8byte) long

실수

  • (4byte) float
  • (8byte) double

정수와 실수는 메모리에 자료형의 크기만큼 공간을 할당하여 데이터를 2진수로 저장한다.

그렇다면, 자바에서 정수와 실수가 어떻게 저장되고 변화될까?

부호비트

우선 정수나 실수 모두 메모리에는 2진수로 변환하여 저장한다.
1byte = 8bit로 표현하는데, 이미지화를 시킨다면
1byte = 00000000와 같이 8개의 비트로 나타낸다.

이 중에서 첫번째 자리의 비트는 부호비트로 사용한다.
부호비트란 양수와 음수를 표현하기 위한 비트로써
0일때는 양수
1일때는 음수
로 표현한다.

부호비트를 제외한 나머지 비트는 정수를 표현할 때 사용하며, 이를 정수부라고 표현한다.

음수를 만드는 방법

사람에게 빼기에 대한 계산식을 적어보라고한다면

이렇게 식을 적고 계산한다.
하지만 컴퓨터에게는 빼기가아닌 이또한 더하기 연산으로 계산한다.

수를 빼는게 아닌 양수에 음수를 더한다고 보면된다.

그러면 빼기 연산자가 실행되는 과정을 살펴보자.

10 - 5라는 식이 있다고 봤을 때
(1byte 기준으로 작성)

  1. 10을 2진수로 변경한다
    1.1 10 -> 00001010
  2. -5를 2진수로 변경한다.
    2.1 5의 2진수 -> 00000101
    2.2 5의 2진수에 1의보수(0은 1로, 1은 0으로)
    00000101 -> 11111010
    2.3 반전시킨 1의 보수에 1비트 가산(2의보수)
    11111011
    3.두 수를 더하기 연산
    00001010 + 11111011

결과: 00000101 -> 5

위처럼 1의 보수와 2의 보수를 이용하여 양수를 음수로 변환 후 더하기 연산으로 값을 도출한다.

단어 뒤집기2 (link)

문제

문자열 S가 주어졌을 때, 이 문자열에서 단어만 뒤집으려고 한다.

먼저, 문자열 S는 아래와과 같은 규칙을 지킨다.

알파벳 소문자(‘a’-‘z’), 숫자(‘0’-‘9’), 공백(‘ ‘), 특수 문자(‘<’, ‘>’)로만 이루어져 있다.
문자열의 시작과 끝은 공백이 아니다.
‘<’와 ‘>’가 문자열에 있는 경우 번갈아가면서 등장하며, ‘<’이 먼저 등장한다. 또, 두 문자의 개수는 같다.
태그는 ‘<’로 시작해서 ‘>’로 끝나는 길이가 3 이상인 부분 문자열이고, ‘<’와 ‘>’ 사이에는 알파벳 소문자와 공백만 있다. 단어는 알파벳 소문자와 숫자로 이루어진 부분 문자열이고, 연속하는 두 단어는 공백 하나로 구분한다. 태그는 단어가 아니며, 태그와 단어 사이에는 공백이 없다.


입력

첫째 줄에 문자열 S가 주어진다. S의 길이는 100,000 이하이다.

예제


출력

첫째 줄에 문자열 S의 단어를 뒤집어서 출력한다.
