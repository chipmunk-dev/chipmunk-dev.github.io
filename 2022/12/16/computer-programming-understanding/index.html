컴퓨터 프로그래밍과 이해

컴퓨터와 프로그래밍의 이해

목차

  • 컴퓨터의 이해
  • 프로그래밍의 이해

컴퓨터의 이해

컴퓨터란 무엇인가?

입력 받은 데이터를 정해진 **규칙**대로 처리해서 처리 결과를 저장하고 출력하는 전자 장치

정해진 규칙대로 → 소프트웨어
입력받은 데이터를 처리하고 그 결과를 출력하는 장치 → 하드웨어

컴퓨터는 소프트웨어와 하드웨어로 구성되어있으며, 이 둘의 상호작용을 통해 동작한다.

소프트웨어

소프트웨어는 하드웨어가 어떻게 동작해야 하는지 규정해놓은 명령어의 집합이다.
이는 크게 시스템소프트웨어 와 응용소프트웨어 로 분류된다.

시스템 소프트웨어
응용 소프트웨어를 실행하기 위한 플랫폼 역활을 수행하며, 하드웨어와 직접적으로 소통한다.

  • 응용소프트웨어의 실행환경
    • Mac, Windows, Linux과 같은 운영체제 및 런타임 환경(Node, JRE)
  • 하드웨어 장치를 관리
    • 키보드 드라이버, 마우스 드라이버, USB 드라이버와 같은 장치 드라이버들

응용 소프트웨어
시스템 소프트웨어를 제외한 모든 소프트웨어를 의미, 애플리케이션/앱 이라고도 하며, 프로그램이라고도 불린다.

하드웨어

컴퓨터를 구성하는 기계적 장치들을 말한다.
CPU, Memory, Graphic 등의 물리적인 장치들이다.
컴퓨터를 구성하는 하드웨어는 중앙처리장치, 기억장치, 입력장치, 출력장치로 분류된다.
그리고 각 장치는 시스템 버스로 연결되어 있다.

중앙처리장치 (Central Processing Unit, CPU)
CPU는 프로그램을 실행, 데이터 처리를 담당하는 핵심 장치이며 프로세서라고도 한다.
제어장치, 연산장치, 레지스터로 구성되어 있다.

  • 제어장치
    • CPU의 동작과 데이터 흐름을 관리한다.
    • 운영체제와 소통하며, 자원을 관리한다.
  • 연산장치
    • 프로그램의 명령어를 수행한다.
    • 코드를 실질적으로 실행시키는 장치
  • 레지스터
    • CPU 안에 있는 작고 빠른 기억장치 → CPU와 가장 가까운 기억장치
    • 연산에 필요한 데이터들을 임시적으로보관하고 제공한다.

기억장치 (Memory Device)
CPU 동작에 필요한 데이터들을 단기 또는 장기적으로 보관한다.
단기적으로 기억하는 것은 주기억장치이며, 장기적으로 기억하는 것은 보조기억장치이다.

  • 보조기억장치
    • 컴퓨터에 필요한 정보를 반영구적으로 저장
    • 큰 용량의 데이터를 저장할 수 있지만 데이터의 일기/쓰기 속도가 상대적으로 느리다.
    • CPU의 연산속도가 엄청 빠른편이나 보조기억장치는 느리기 떄문에 직접적으로 소통하지 않는다.
    • SSD(Solid State Drive), HDD(Hard Disk Drive)가 있다.
  • 주기억장치
    • 컴퓨터가 프로그램을 수행하는 동안 필요한 모든 데이터들을 저장
    • 보조기억장치와 같이 큰 용량을 저장할수는 없지만 데이터의 읽기/쓰기 속도가 빠른편이다.
    • 빠른 연산 속도로 인하여 CPU와 직접적으로 소통한다.
    • RAM(Random Access Memory), ROM(Read Only Memory)가 있다.

캐시메모리
캐시메모리는 CPU가 사용한 데이터 중에서 재사용성이 높은 데이터를 주기억장치로부터 미리 저장한다.
주기억장치가 빠르다지만 CPU만큼은 아니기에 캐시메모리를 통해 데이터들을 좀더 빠르고 효율적으로 저장하고 가져올 수 있다.

기억장치의 계층구조
레지스터 → 캐시메모리 → 주기억장치 → 보조기억장치
우측으로 갈수록 메모리의 가격이 저렴하며, 용량을 많이 저장할 수 있다.
하지만 반대로 속도는 느려진다.

입력 장치 (Input Device)
사용자로부터 컴퓨터에 데이터를 입력받는 장치 → 키보드, 마우스,마이크 등

출력 장치 (Output Device)
사용자에게 사람이 감각할 수 있는 형태로 데이터를 출력해주는 장치 → 모니터, 스피커, 프린터 등

시스템 버스(System Bus)
하드웨어 장치간의 통로
각각의 장치를 물리적으로 연결한 전선을 의미하기에 시스템 버스도 하드웨어에 포함된다.

  • 데이터 버스
    • 하드웨어 간에 데이터를 전달하는 통로
  • 주소 버스
    • 데이터가 어디에 도착해야 하는지에 대한 정보를 전달하는 통호
  • 제어 버스
    • CPU의 제어장치가 생성한 제어 신호를 다른 장치로 전달하는 통로

프로그래밍의 이해

컴퓨터에게 문제 해결을 맡기는 커뮤니케이션

프로그램을 만드는 과정, 특정 목적을 달성하기 위해 설계된 알고리즘을
프로그래밍 언어를 사용하여 코드로 작성하는 과정

프로그램

프로그래밍을 통하여 만들어진 결과물
즉, 어떠한 목적을 달성하기 위한 컴퓨터 명령문의 집합체

프로그래밍 언어

컴퓨터는 사람들의 언어를 이해하지 못한다.
그러기에 컴퓨터가 이해할 수 있는 언어로 설명해주어야 한다.
컴퓨터는 기계어를 사용한다. 사람들은 사람들만의 언어를 사용하는데,
이 둘의 매개해주는 언어가 프로그래밍 언어다.
사람들의 명령어를 이해하고 이를 통하여 컴퓨터를 동작하게 한다.

컴파일

프로그래밍 언어는 사람의 언어와 컴퓨터 언어를 매개하지만,
컴퓨터는 프로그래밍 언어로 작성된 소스코드를 이해하지 못한다.
이를 컴퓨터에게 이해시키기 위해 코드를 변환하는 작업을 하는데 이를
**컴파일(Compile)**이라고 한다.
자바는 컴파일을 통하여 소스코드를 byte코드로 변환하여 JVM에게 던져준다.

빌드

작성한 소스코드를 실행할 수 있는 결과물로 변환시키는 과정을 빌드라고한다.
컴파일은 빌드과정의 일부이다. 컴파일을 마친 코드는 기계어 코드로 변환하고 이를 통하여 실행할 수 있는 .exe, .msi등 확장자를 가진 실행 파일로 만든다. 이런 빌드가 완료되어야 실행 가능한 프로그램이 완성된다.

통합 개발환경 (IDE)

Integrated Development Environment(이하 IDE)는 코드를 작성할 때 사용하는 프로그램으로, 프로그래밍에 필요한 모든 작업을 하나의 프로그램으로 처리할 수 있도록 도와준다.
코드를 작성하고 편집할 수 있으며, 디버깅과 빌드도 IDE에서 가능하다.
대표적으로 Visual Studio Code, IntelliJ, Eclipse등이 있다.
